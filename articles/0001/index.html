Ever since the term was introduced by Benoit B. Mandelbrot, "fractals" have awed so many people with their odd combination of simplicity and complexity. In this article I specifically talk about fractals in the complex plane, but fractal geometry and its derivatives are a fascinating part of math going much beyond what I detail here, so definitely get yourself interested in that. Before we begin, let's figure out exactly what a fractal is. Dictionary.com defines the word fractal to mean "a geometrical or physical structure having an irregular or fragmented shape at all scales of measurement between a greatest and smallest scale such that certain mathematical or physical properties of the structure, as the perimeter of a curve or the flow rate in a porous medium, behave as if the dimensions of the structure (fractal dimensions) are greater than the spatial dimensions." This is quite wordy, sure, but to get a better idea of what a fractal is, let's look at some examples. But as with any object, without some deeper meaning, a lot of beauty is lost. Let's delve into how the Mandelbrot set is generated. First, let's consider the name. The Mandelbrot set is really just that: a set. That means it's effectively a list of points. When we color these points black and the rest of the points white, we get the figure from above. But how is this set determined? To understand the construction of the Mandelbrot set, we need to know about complex numbers. Complex numbers, of course, are an incredibly rich subject, and they have applications in physics, statistics, and much more. In essence, complex numbers are based off an "imaginary number" called i, defined so that \(i^2=-1\). Like any other number, the conventional rules for arithmetic addition and subtraction work with i. For example, \(2i+2i=4i\) and \(5i-2i=3i\). By the definition of i, we know that \(i^2=-1\). But what's the value of, say, \((3i)^2\)? Well, we can treat this expression as \((3 \cdot i)^2 = 3^2 \cdot i^2\). Using our definition of i, we thus know that \((3i)^2=9\cdot-1=-9\). How about complex numbers? Well, when we add them, we just add their real components and their imaginary components in turn. For example, \((2+3i)+(4+9i)=(2+4)+(3+9)i=6+12i\) and \((2+3i)-(4+9i)=(2-4)+(3-9)i=-2-6i\). Multiplication of complex numbers is a tad tricker, but we just use the distributive property. For example, \((2+3i)\cdot(4+9i)=8+18i+12i-27=-19+30i\). We have these lovely complex numbers now. But is there a way to visualize them? Sure thing! We can think of these complex numbers as points on a plane, sort of like a Cartesian coordinate system. The number \(a+bi\) corresponds to the point \((a,b)\) on the coordinate plane. Of course, this is just a very basic introduction to complex numbers and the idea of the complex plane, so if you haven't learned about them yet and really want to understand what will be going on in the rest of this article, reading about complex numbers and the complex plane may be helpful. With that out of the way, let's figure out what the Mandelbrot set is about! Define a function \(f_c(z)=z^2+c\) over the complex numbers. Then the Mandelbrot set is the set of all numbers \(c\) for which the function \(f_c(z)\) does not diverge when iterated from \(z=0\). In other words, \(f_c(f_c(f_c\cdots (z)))\) stays bounded. Let's test if the number \(0.2+0.5i\) is in the Mandelbrot set. We start with \(f_c(0)\), where \(c=0.2+0.5i\). In the next iteration, we do \(f_c(0.2+0.5i)=-0.01 + 0.7i\), then in the next we do \(f_c(-0.01 + 0.7 i)\), etc. The results of these iterations are shown ::fig::images/complextable1.png::Iterations after c=(0.2+0.5i)::Figure 2: The numbers produced by iteration of \(f_c\).:: It seems \(c=0.2+0.5\) does not diverge to infinity; it stays bounded. Thus, it is in the Mandelbrot set. Overlaying a point \(c=0.2+0.5i\) on the original image, we can see that, indeed, the point lies in a black region. (The x and y axes are also indicated.) ::fig::images/mandelbrot2.png::Mandelbrot Set (Real axis: [-2.5,1.5], Imaginary axis: [-2,2], 500 iterations.)::Figure 3: The Mandelbrot set, overlayed with point \(c=0.2+0.5i\).:: This repeated calculation is done for every point - or, at least, every pixel in the image - and if the calculation stays bounded to a reasonable number of iterations, we can assume that it probably will stay bounded forever. Now, let's see how these images were generated in code. My language of choice was C++, but any language with a complex number type will do with this method. First, we define our constants. These are 1. the image's height and width, 2. the bounds on the real axis, 3. the bounds on the imaginary axis, 4. the number of iterations to execute, and 5. the "escape radius," or the magnitude of the complex number where it is treated as if it will go to infinity. For taking a picture of the Mandelbrot set, the following settings are suitable, but you may want to adjust them for greater accuracy. For example, if the fractal boundary is too smooth, increasing the iteration count might help. For most fractals, an escape radius of 2 is too small, so set it to some number like 50 for fractals besides the Mandelbrot set.
